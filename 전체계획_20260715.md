## 1. 목표와 현재 위치

**목표**: 라벨링 업체가 즉시 쓸 수 있는 Instance 크기·Point 기준 확정 (itivai 2~4순위) + YOLO26x-seg@1280 기준 학습 최적화.

| 순위 | 항목 | 목표일 | 상태 |
|---|---|---|---|
| 1순위 | Point 수 기준 (itivai 담당) | 6/12 | 완료 (인스턴스당 50~100pt 가이드) — 최종 가이드 문서 수신 대기 |
| 2순위 | 무의미 Instance 최대 크기 | **7/24 재검증 공유** | 잠정 기준 회신(7/8) → 확인요청 3건 대응(7/14) → **재학습 7건 진행 중** |
| 3순위 | 이미지 크기·Point 수 균형 | 7/31 스크리닝, 8/4 최종 | 코드 준비 완료 (exp3) |
| 4순위 | 유의미 Instance 최소 크기 고도화 | 8/21 중간, 9/4 최종 | 코드 준비 완료 (exp4) + 신규 근거 확보 |

## 2. 확정·제안 기준 (2순위 잠정안 v2)

1. **기본 컷오프**: √(폴리곤 면적) 10px@1280 = **원본 30×30px(900px², 면적비 0.011%) 미만 라벨링 생략** — exp5 전수 시뮬레이션 근거(원본 30px 이상은 stride4 GT 형상 보존율 IoU≥0.5가 97%+).
2. **세장형 예외 (개정)**: 면적 미달이어도 **긴 변 ≥ 원본 72px 그리고 평균 두께(면적/긴 변) ≥ 원본 6px**이면 라벨 유지. (기존: 긴 변 조건만. 개정 근거: 두께 6px 미만은 stride4 GT가 52~74% 조각남 — 심지훈 "철근 소실" 관찰의 정량 재현.)
3. 효과: 라벨링 대상 46.5%→44.3%, "라벨링했는데 GT가 뭉개지는" 비율 5.9%→2.3%.
4. **주의**: 기준은 학습용 필터로도 구현돼 있으므로(코드 필터) 라벨 원본은 보수적으로 유지 가능 — 삭제가 아니라 생략 기준.

## 3. 진행 중 실험 (node002, 2026-07-15 19:30 기동)

YOLO26x-seg, imgsz=1280, batch=4, 100ep, ultralytics 8.4.95 + torch 2.4.1+cu118 (드라이버 520 호환).

| GPU | 실험 | 조건 | 목적 |
|---|---|---|---|
| 0 | exp2 | base (무필터) | 대조군 |
| 1 | exp2 | cut8 (원본 24px) | 이론 하한 |
| 2 | exp2 | cut10 (원본 30px) | **제안 기준** |
| 3 | exp2 | cut12 (원본 36px) | 중간 |
| 4 | exp2 | cut16 (원본 48px) | 검증 상한 |
| 5 | exp6 | cut10_noexc (예외 없음) | 세장형 예외 자체의 기여 |
| 6 | exp6 | cut10_w2 (두께≥6px 예외) | **개정 예외안** 검증 |
| node005 A100 | exp7 | cut10 + **mask_ratio=2** (batch 4) | 라벨링 기준 대신 **학습 설정**으로 얇은 객체 GT 보존(두께 하한 6px→3px). ⚠️ **실측 발견: mask_ratio=1은 불가** — 피크 메모리가 배치가 아니라 단일 이미지 인스턴스 수에 비례(이미지당 [N×1280²] 마스크 텐서)해 batch 2에서도 A100 80GB OOM(66+17GB). 실용 대안 ratio 2로 batch 4(=cut10과 동일 배치) 진행, 57.9GB 안착. node002 GPU 7은 예비 |

- **판독 기준**: 공통 val(무필터 라벨) mask mAP50 기준 비교(자기 val 비교는 왜곡) + rebar/small pipe/pipe/square pipe/mesh 클래스 AP50.
  - 채택 규칙: ① exp2에서 mAP 최고이면서 세장형 AP가 유지되는 컷 ② exp6에서 cut10_w2 ≥ cut10이면 두께 예외 채택.
- **모니터링**: 결과 CSV(exp2_results.csv/exp6_results.csv) 행 수를 세션이 폴링 → 조건별 완료/에러 시 **Claude 앱 푸시** (Telegram 폐기, 2026-07-15).
- 예상 소요: 조건당 ~15-24h (7/17 내 완료 예상 → 7/24 공유 여유).

## 4. 후속 실험 계획

### 4a. exp3 — 3순위 그리드 (7/31 스크리닝 → 8/4 최종)
- imgsz {960, 1280, 1600, 1920} × RDP 단순화 eps {0, 4, 8}px@1280, 컷오프 8px 고정, 60ep 스크리닝 → 상위 2~3 조합 300ep 확정.
- 측정: 크기별 Box/Mask mAP, 추론 속도, VRAM. **특별 판독**: rebar·small pipe AP의 입력 크기 민감도 — 1280의 구조적 한계(rebar IoU≥0.5 35.7%)가 1600/1920에서 얼마나 풀리는지 → 세장형 두께 하한 재조정 근거.
- 실행: exp2/exp6 종료 후 같은 방식으로 node002 병렬 (12조합 → GPU 6장 × 2회전).

### 4b. exp4 — 4순위 클래스별 최소 크기 (8/21 중간 → 9/4 최종)
- exp2 best 가중치로 val 추론 → 클래스×크기구간 recall → 클래스별 "recall 급락 최소 구간" 도출.
- 신규 반영: 작년 라벨 품질 이슈(장준희 4/3 메일 — **rebar 덩어리 라벨링 다수**, Deck reinforcement steel 오라벨 1,000장+, handler 과대 라벨) → 클래스별 기준에 "개별 vs 묶음" 정의 포함.

### 4c. 학습 방법 개선 백로그 (라벨링 기준과 직교 — exp2/6 결과 본 뒤 우선순위 결정)
| 실험 | 내용 | 근거 | 비용 |
|---|---|---|---|
| **mask_ratio=1** | GT를 다운샘플하지 않고 seg loss 계산 (하이퍼 1개) | 얇은 객체 GT 소실의 직접 원인 제거 — "라벨링을 더 시키느냐 vs 학습 설정을 바꾸느냐" 판가름 | 메모리 ~16배(배치 축소로 흡수), 1런 |
| copy_paste=0.3 | 소형·희소 클래스 오버샘플링 증강 | 클래스 불균형(small pipe 12만 vs pipe 3.3천) + Kisantal'19 소형 seg +9.7% | 1런 |
| scale=0.3 | 축소 증강 하한 완화 (0.5×→0.7×) | 컷 경계 객체가 학습 중 GT 뭉개짐(exp5 _s0.5 표) | 1런 |
| overlap_mask=False | 겹침 인스턴스 독립 마스크 학습 | 적층 장면에서 작은 객체가 큰 객체에 먹힘 | 메모리↑, 후순위 |

### 4d. 라벨 QC 자동화 (라벨링 업체 납품 검수용 — 착수 가능)
- `prepare_yolo_dataset.py` 확장: 크기 기준 위반 / 폴리곤 자기교차·중복 / 클래스명 매핑 실패 / Cargo 영역 밖 / 두께 미달 세장형 자동 리포트.
- 신규 라벨 샘플 수신 시(1순위 가이드 문서와 함께 요청해 둠) 즉시 적용.

## 5. 라벨링 가이드 권고 (itivai 전달용 — 크기 기준 외)

1. **rebar 덩어리 금지**: 개별 식별+크기 통과 객체만 개별 인스턴스, 뒤엉킨 다발은 묶음 1개로 — 이미지 내 일관성 필수 (작년 실측 이슈 재발 방지).
2. **겹침 경계 정확히**: 학습이 겹침 마스크를 면적순 덮어쓰기(overlap_mask)로 처리 → 보이는 영역만, 경계 침범 금지. handler는 크레인 제외.
3. **경계 디테일 상한**: 원본 12px(proto cell 1개) 미만 굴곡은 학습에 반영 안 됨 — 공수 절약. 단 **얇은 객체의 두께는 정확히** (6px 경계가 keep/drop을 가름).
4. **컷오프 일관성**: 한 이미지 안에서 기준 통일 (같은 크기를 하다 말다 금지 — 모순 신호 방지).
5. **Count 정의 합의 (계약적)**: 컷오프 적용 시 Count Acc/Area Ratio의 분모(정답 개수) 정의가 바뀜 — 7/24 회신에서 명시 합의 필요.
