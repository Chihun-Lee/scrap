# 철스크랩 라벨링 방안 정리 (ver2 준비)

ITV AI(아이티브에이아이) 심지훈 선임이 보낸 **라벨링 기준 연구 요청서(수정본 0528)** + 후속 메일 2통을 종합한 현재 라벨링 방안 정리.
정리 기준일: 2026-06-03.

---

## 0. 우리(재료연구원)의 위치
- **발주/협업 주체**: ITV AI(심지훈 선임) — 라벨링 업체 관리 + 일부 연구 직접 수행
- **우리 = 재료연구원(KIMS)** — 라벨링 기준 수립을 위한 **연구 항목 2·3·4순위 담당**
- 라벨링은 **외주 업체**가 수행. 우리는 업체가 따를 **기준(가이드라인)을 도출**하는 역할.

---

## 1. 사업 개요
- **목표**: 철스크랩 **Instance Segmentation 라벨링 가이드** 기준 수립
- **라벨링 기간**: 2026-06-17 착수 → 2026-10월 마무리
- **타깃 모델**: **YOLO26x-seg** (※ ver1은 YOLO11-seg, 이번엔 YOLO26 계열)
- **진행 철학(중요)**: 단발성 분석이 아니라 순환 구조
  ```
  초기 기준 수립 → 라벨링 적용 → 모델 성능 검증 → 기준 보완 (반복)
  ```
  기준이 바뀌면 **변경 사유 + 실험 결과 + 성능 변화 + 라벨링 영향**을 함께 공유.

---

## 2. 라벨링 운영 원칙
- **기존 라벨 최대한 유지** — 초기 단계에서 대량 삭제/재작업 지양.
- **누락·추가 필요 객체를 "보완(추가 라벨링)"하는 방식**이 효율적.
- 즉 초기 기준은 "무엇을 지울까"가 아니라 **"무엇을 추가로 라벨링할까"의 범위**를 정하는 방향.
- 단, 라벨링 진행 중 모델 성능에 **악영향을 주는 과소 객체·불명확 객체·과도한 Polygon**이 확인되면 최종 기준표에 반영.

## 3. 클래스 목록 변경사항 (킥오프 후속, 메일①)
ver1의 의미 혼선 문제를 줄이기 위한 분류 규칙 변경:

### 3-1. Unknown-something 활용방안 변경
| 구분 | 적용 조건 |
|------|-----------|
| **기존** | (i) 특정 클래스로 보이나 **불확실**한 경우 → (2번 방식으로 전환) <br> (ii) 제품은 식별되나 **정의된 클래스 목록에 없는** 경우 |
| **변경** | **(ii)만** Unknown으로 분류 — 즉 *제품은 식별되나 클래스 목록에 없는 경우*에만 사용. <br> 불확실한 경우엔 Unknown을 쓰지 않음. |

### 3-2. 형상·현장 분류기준에 따른 "대표 클래스" 설정
- 유사 클래스가 **경합**할 때 → **대표 클래스 하나로 분류**.
- 예: 둥근 파이프 형태가 "배관"인지 "아연도금 강관"인지 경합 → **"배관"으로 분류**.
- → ver1에서 `Unknown-Panel→structure steel` 식으로 외형 다른 것이 섞이던 문제를 줄이는 규칙.

## 4. 연구 우선순위 · 담당 · 일정 (PDF + 메일② 반영)
메일②에서 **담당과 일정이 조정**됨. 특히 **1순위는 ITV AI가 직접 수행**(착수 전 선확보 필요), 2~4순위가 재료연구원 몫.

| 순위 | 연구 항목 | 기준 해상도/조건 | 목표 일정 | 담당 |
|:----:|-----------|------------------|:---------:|------|
| **1** | 모델 학습에 필요한 **러프한 Polygon Point 수 기준** 도출 | 입력 **1280** 기준 | **6/12** | **ITV AI** |
| **2** | 학습에 무의미한(=충분히 크지 않은) Instance 구분용 **러프한 (최대) 크기 기준** → "충분히 큰 Instance 우선 라벨링" | 원본 **3840×2160** | **6/26** | **재료연구원** |
| **3** | 학습·추론 **성능·속도 균형** 입력 이미지 크기 + Point 수 조합 | 960/1280/1600/1920 등 | **8/4** | **재료연구원** |
| **4** | **학습에 유의미한 Instance 최소 크기 기준** 고도화 | 원본 **3840×2160** | **9/4** | **재료연구원** |

> 해석: **2순위(러프, 착수용) ↔ 4순위(고도화)** 는 동일한 축(소형 Instance 크기 컷오프)을 *먼저 거칠게 → 나중에 정밀하게* 잡는 관계.
> 1순위(Point 수)·3순위(크기×Point 조합)는 GPU/속도/품질 트레이드오프 축.

## 5. 순위별 "확인 요청 사항" (업체·연구 산출물 체크리스트)

### 1순위 — Point 수 기준 (담당: ITV AI, 참고용)
*문제의식: 4K 원본 기준 Polygon이 과도하게 세밀 → 이미지당 Instance·Point 수 과다 → 학습 시 GPU 메모리 폭증.*
- 입력 1280 기준 **YOLO26x-seg 학습 가능 여부**
- **OOM 안 나는** Polygon Point 수 기준
- **Instance당 최대 Point 수** / **이미지 1장당 총 Point 수** 기준
- **Point 단순화 수준별** Mask 품질 변화
- Point 단순화 수준별 학습 가능 여부 & GPU 메모리 사용량 변화
- → 업체가 Polygon 작성/보정 시 참고할 **Point 수 상한 기준**으로 활용

### 2순위 — 충분히 큰 Instance 우선 라벨링 (담당: 우리, 6/26)
*문제의식: 객체가 겹침·절단·구김·파손으로 일부만 보임. 누락 객체·소형 객체 추가 여부 판단 어려움.*
- 3840×2160 기준 **우선 추가 라벨링해도 무리 없는 충분 크기 Instance 구간**
- **이미지 전체 면적 대비 Instance 면적 비율** 기준
- 객체 크기 기준 적용 시 **추가 라벨링 대상 Instance 수 변화**

### 3순위 — 이미지 크기 × Point 수 조합 최적화 (담당: 우리, 8/4)
- 입력 이미지 크기별 **Box mAP / Mask mAP** 변화
- 입력 크기별 **추론 속도** 변화
- 입력 크기별 **GPU 메모리** 변화
- **Point 수 단순화 수준별** 성능 변화
- **속도·성능 균형점**이 되는 (이미지 크기 × Point 수) 조합

### 4순위 — 학습 유의미 최소 크기 고도화 (담당: 우리, 9/4)
- 3840×2160 기준 **학습에 의미 있다고 볼 수 있는 최소 Instance 크기**
- **클래스별** 최소 Instance 크기 차이
- **겹침·절단·구김·파손** 객체의 유지 또는 추가 라벨링 기준
- 기준 적용 시 모델 성능·라벨링 효율 변화

## 7. 열린 질문 / 확인 필요
- **1순위 결과 전달**: ITV AI가 6/12 도출한 Point 수 기준을 우리가 받아 2~4순위 실험의 입력으로 쓰는지 (특히 3순위는 Point 수가 변수).
- **YOLO26x-seg 가용성**: 실제 학습에 쓸 버전/환경 — 클러스터에 설치 가능한지.
- **데이터 기준**: 2·4순위가 "원본 3840×2160 기준"인데, 우리 실험은 ver1 datasets(이미 19클래스 remap·8px 필터 적용)을 쓸지, 원본 LabelMe를 다시 받을지.
- **클래스 불균형**: 4개 연구 항목엔 없음 — 별도 제안 필요 여부.
- **"면적 비율" 정의**: 2순위 "이미지 면적 대비 Instance 면적 비율"의 기준 단위(px², %).
