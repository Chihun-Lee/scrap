# 03. 소형 객체 / 최소 Instance 크기 / Tiling — 2·4순위 근거자료

> 목적: (a) "충분히 큰 instance"(라벨링 착수용), (b) 최소 학습가능 instance 임계 — 둘 다 3840×2160 원본 기준, YOLO11-seg 입력 1280.
> 작성: 2026-06-03.

---

## Part 1 — 검증된 사실 (수식의 토대)

### COCO small/medium/large 면적 정의 (cocoapi source 확인)
평가 이미지상 segmentation/box 픽셀 면적 기준:
- **Small**: `[0², 32²]` = **0–1,024 px²**
- **Medium**: `[32², 96²]` = **1,024–9,216 px²**
- **Large**: `> 96²` = **> 9,216 px²**
출처: cocoapi `cocoeval` `areaRng` (cocodataset/cocoapi #662, #297). small ≤ 32×32px, medium ≤ 96×96px.

### YOLO11-seg feature-map stride (검증)
- detection head: **P3(stride 8), P4(16), P5(32)**. 640 입력 → 80/40/20 grid. 최소 detection stride 8 (stock YOLO11엔 P2/stride-4 head 없음).
- **mask prototype: stride 4** (640 입력서 160×160) — detection grid보다 고해상도.

---

## Part 2 — 스케일 팩터 추론 (명시적 유도)

YOLO는 long side를 입력 크기로 리사이즈 + letterbox. **3840×2160 → 1280: s = 1280/3840 = 1/3.**
→ **원본 1px → 입력 1/3px**, 즉 **입력 1px = 원본 3px**. 원본 side `L` → 입력 `L/3`.

- **detection head(P3, stride 8) 1 cell 채우려면** 입력 ≥8px → 원본 **24px** (~576 px²). 미만이면 box 분기가 사실상 anchor 불가.
- **mask prototype(stride 4) 1 cell 채우려면** 입력 ≥4px → 원본 **12px** (~144 px²). 미만이면 의미있는 마스크 못 받음.
- COCO "small" 경계 32 입력px → 원본 **96px**(~9,216 원본px²). COCO "medium" 경계 96px → 원본 **288px**.
→ **원본에서 한 변 < ~96px인 instance는 입력 1280서도 여전히 "COCO-small"** = detector가 가장 약한 영역.

> v1 문제(">절반이 4K→640서 8px 미만")와 정확히 일치. 640서 s=1/6 → 8입력px=원본 48px. **1280(s=1/3)으로 가면 같은 8입력px floor가 원본 24px로 절반** — 입력 해상도만으로 2× 개선.

## Part 3 — 참고문헌

**1. Microsoft COCO** — Lin et al. — ECCV 2014 — https://arxiv.org/abs/1405.0312
small/medium/large bin(32²,96²) 정의. → COCO bin을 *어휘*로 사용. s=1/3 역산 시 "COCO-small" ≈ 원본 < ~96px side = "이하면 성능 저하 예상" 구간 후보.

**2. FPN** — Lin et al. — CVPR 2017 — https://arxiv.org/abs/1612.03144
multi-scale pyramid; 소형은 최고해상도 level에서. stride→최소해상가능객체 연결 확립. 1280서 hard floor ~24px(detect)·~12px(mask).

**3. SAHI (Slicing Aided Hyper Inference)** — Akyon et al. — IEEE ICIP 2022 — https://arxiv.org/abs/2202.06934
고해상 이미지를 겹치는 tile로 추론+fine-tune → 소형이 입력 대비 커짐. aerial서 +6–14% AP.
→ **높은 min-size 임계의 가장 실질적 대안**. 4K를 1280 crop으로 tiling하면 s≈1, 원본 최소 학습크기가 ~3× 작아짐. 소형 instance 버리기 전에 tiling 강력 고려.

**4. Augmentation for Small Object Detection** — Kisantal et al. — 2019 — https://arxiv.org/abs/1902.07296
소형 copy-paste oversampling으로 small AP +9.7%(seg). → 붕괴한 minority 클래스에 유효. 희귀 클래스는 임계 근처라도 보존+oversample 권장 = 희귀 클래스 min 임계를 *낮게*.

**5. Scale Match / TinyPerson** — Yu et al. — WACV 2020 — https://arxiv.org/abs/1912.10664
객체 <20px 빈번한 벤치마크. ~20px(학습 해상도 기준)가 detector 저하 시작 영역 확인. 우리 ~24px detection floor가 바로 이 경계 → 최소 학습 임계를 cell-floor보다 약간 위로.

**6. SODA survey** — Cheng et al. — TPAMI 2023 — https://arxiv.org/abs/2207.14096
COCO "small" 아래 더 미세한 sub-bin 정의. 단일 pass/fail 대신 "tiny/small/OK" 다단 임계 설계에 참고.

**7. QueryDet** — Yang et al. — CVPR 2022 Oral — https://arxiv.org/abs/2103.09136
고해상 feature를 희소 query → small mAP +2.0, 3× 가속. 고해상/미세 stride 유지가 소형 회복의 핵심 → 1280(또는 tiling)·stride-4 prototype 의존 지지.

**8. Finding Tiny Faces** — Hu&Ramanan — CVPR 2017 — https://arxiv.org/abs/1612.04402
3px vs 300px는 근본적으로 다른 cue. 수 px 이하는 아키텍처 무관 신호 부족 → single-cell limit 근처 hard floor + upsampling/tiling 정당화.

**9. Is Annotating Small-size Instances Worth Its Cost?** — Hosoya et al. — 2024 — https://arxiv.org/abs/2412.05611
**가장 직접적 근거.** 소형 instance를 *제외*하고 학습해도 test-time upscaling과 결합 시 full 데이터와 비슷한 성능. → 최소 instance를 안 찍어도 안전, "라벨링할 만큼 충분히 큼" 임계를 네트워크 floor보다 넉넉히 위로 잡아도 정확도 손실 적음.

**10. On the Importance of Large Objects in CNN Detection** — https://arxiv.org/pdf/2311.11714 — [UNVERIFIED: 저자/venue 미확인]
대형 객체가 학습 신호 지배. 소형은 어차피 기여 적음 → 비자명한 min 임계 강화(보조 근거).

## Part 4 — 종합 권고 (추론, 위 근거 기반)

**3840×2160 원본, YOLO11-seg 입력 1280 (s=1/3):**

| 임계 | 원본 크기 | 근거 |
|------|-----------|------|
| **Hard floor — 학습 무의미** | side **< ~12px** (~144px²) | < 1 stride-4 mask cell, 마스크 불가 (검증 stride 수식) |
| **Detection-anchor floor** | side **< ~24px** (~576px²) | < 1 P3 stride-8 cell (검증 stride 수식) |
| **정제된 최소 학습 임계 (4순위)** | side ≈ **24–48px** | single-cell floor 위 + TinyPerson "<20px 저하" (ref 5,8) |
| **"충분히 큼 — 라벨링 가치" (2순위 착수)** | side ≳ **~96px** (~9k px² 원본) | 입력서 COCO-medium+ 영역, ref 9: 이하 소형은 skip 가능 |

**모든 수치를 바꿀 수 있는 구조적 변수 2가지:**
1. **tiling/SAHI(ref 3)** 채택 시 s→~1 → 모든 원본px floor가 ~3× 축소(hard floor ~4px, 착수 ~32px). ~200 dense instance엔 강력 권장.
2. **희귀/minority 클래스**(v1 붕괴): 전역 cutoff 대신 *낮은* min 임계 + copy-paste oversample(ref 4).
