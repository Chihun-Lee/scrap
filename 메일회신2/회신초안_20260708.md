## 1. 2순위 — 학습에 무의미한 Instance의 러프한 최대 크기 기준 (진행 상황)

### 1-1. 진행 내용

① (완료) **전체 라벨 데이터 정량 분포 분석** — 기존 라벨 전체(Train 417,218 / Val 84,159 인스턴스, 2,515장)를 대상으로 1280×1280 입력 환산 크기 분포와 컷오프별 제외 비율을 산출했습니다.
② (완료) **기존 1024 기준 실험의 1280 기준 재해석** — 지난 6/23 공유드린 최소 픽셀 필터 실험(10~16px 스윕, mAP 0.787→0.901)을 1280 입력 기준으로 환산해 이론 하한과 교차 검증했습니다.
③ (진행 중) **YOLO26x-seg + 1280 학습 재검증** — 필터 수준별 재학습 스윕 코드 준비를 완료했고, 원내 GPU 클러스터에서 순차 실행 예정입니다.

### 1-2. 잠정 기준 (러프, 1280×1280 기준)

**"학습에 무의미한 Instance의 러프한 최대 크기"는 1280 입력 기준 √면적 8px — 즉 원본 3840×2160 기준 약 24×24px(576px², 이미지 전체 면적 대비 약 0.007%) — 로 제안드립니다.** 이 미만 객체는 라벨링을 생략해도 학습 손실이 없다고 판단됩니다.

근거는 다음 세 가지가 서로 정합합니다.

- **모델 구조 하한**: YOLO 계열의 최소 검출 그리드(P3, stride 8)는 1280 입력에서 8px, 마스크 prototype(stride 4)은 4px입니다. 1280 입력에서 8px(원본 24px) 미만 객체는 검출 그리드 1칸에도 미달해 구조적으로 학습 기여가 어렵습니다.
- **기존 실험 결과**: 6/23 공유드린 스윕에서 필터를 높일수록 mAP가 단조 증가(0.787→0.901)했고, 최적점(16px@1024, bbox 짧은 변 기준)은 원본 환산 약 29px로 위 이론 하한(원본 24px)과 같은 구간에 있습니다.
- **공개 데이터셋 선례**: COCO는 32×32px 미만을 small로 별도 취급하고, BDD100K는 면적 50px² 미만 라벨을 자동 삭제하며, KITTI는 높이 25px 미만 객체를 평가에서 제외합니다. 국제 데이터셋들의 최소 크기 컷과 유사한 수준입니다.

### 1-3. 요청하신 산출물 기준 정리

**(a) 면적 비율 기준 및 (b) 기준 적용 시 대상 Instance 수 변화** (기존 라벨 분포 기준, Train/Val 편차 1%p 미만):

| 컷오프 (1280 기준 √면적) | 원본 환산 | 면적 비율 | 제외되는 Instance 비율 |
|---|---|---|---|
| 6px 미만 | 18×18px (324px²) | 0.004% | 32.8% |
| **8px 미만 (제안)** | **24×24px (576px²)** | **0.007%** | **48.0%** |
| 10px 미만 | 30×30px (900px²) | 0.011% | 59.2% |
| 12px 미만 | 36×36px (1,296px²) | 0.016% | 67.6% |

즉 제안 기준(8px@1280) 적용 시 **추가 라벨링 대상이 약 48% 감소**하여 라벨링 공수가 절반 수준이 됩니다.

**(c) 충분히 큰(우선 추가 라벨링) Instance 구간**:

| 구간 (1280 기준 √면적) | 원본 환산 / 면적 비율 | 권고 | 비중 |
|---|---|---|---|
| 16px 이상 | 48×48px 이상 / 0.03% 이상 | **우선 추가 라벨링 (안전 구간)** | 21.3% |
| 8~16px | 24~48px / 0.007~0.03% | 여력 시 라벨링, 4순위에서 클래스별 확정 | 30.7% |
| 8px 미만 | 24px 미만 / 0.007% 미만 | 생략 가능 (학습 무의미) | 48.0% |

**세장형 객체 예외 규칙(중요)**: 파이프·철근류는 면적은 작아도 학습에 유의미한 경우가 많습니다(6/23 정성 비교에서 확인된 "작거나 긴 파이프 소실" 문제와 동일 맥락). 실제 분포에서도 파이프류(Pipe_2, 99,414개)와 철근류(Coiled reinforcing bar, 40,129개)는 크기 중앙값 자체가 컷오프 부근(√면적 5.1~5.4px@1280)입니다. 따라서 **면적 기준 미달이어도 bbox 긴 변이 원본 기준 72px(1280 기준 24px) 이상인 가늘고 긴 객체는 라벨링을 유지**하는 보조 규칙을 함께 제안드립니다(이 예외로 전체의 약 2.3%, 11,500여 개 인스턴스가 추가 유지됩니다). 세장형 클래스별 정밀 하한은 4순위에서 확정하겠습니다.

### 1-4. 추가 검토(1280 학습 재검증) 예상 소요

YOLO26x-seg + 1280 조건 필터 스윕(무필터/6/8/10/12px, 5개 조건) 재학습으로 위 잠정 기준을 정량 확정하는 데 **약 2~3주**가 소요될 것으로 예상되며, **7월 24일(금)** 결과 공유를 목표로 하겠습니다. (참고: YOLO26x-seg 1280 학습은 파라미터 62.8M, batch 4 기준 GPU 메모리 약 17~25GB로 추정되어 원내 95GB 노드에서 무리 없이 가능할 것으로 사전 검토했습니다.)

## 2. 3순위 — 학습·추론 성능과 속도의 균형적 이미지 크기 및 Point 수 (진행 계획)

**8월 4일까지 진행 가능합니다.** 계획은 다음과 같습니다.

- **실험 매트릭스**: 입력 이미지 크기 {960, 1280, 1600, 1920} × Point 단순화 3수준 {원본 유지 / 중간(정점 간격 ≈ 원본 12px) / 강한 단순화(≈ 원본 24px)} — 총 12개 조합
- **측정 항목** (요청서 기준): 크기별 Box mAP·Mask mAP, 추론 속도(ms/장), 학습 GPU 메모리 사용량, Point 단순화 수준별 성능 변화, 그리고 소형 객체 구간 AP(2·4순위와 연동)
- **진행 방식**: 단축 에폭 스크리닝으로 12개 조합 전수 비교 후, 균형점 후보 2~3개 조합만 전체 학습으로 확정
- **사전 분석 참고**: Point 수는 GPU 메모리보다는 라벨 용량·전처리 속도에 주로 영향을 주며(메모리 주동인은 이미지당 인스턴스 수와 입력 크기), 문헌상으로도 폴리곤 정점 수는 인스턴스당 약 64점 수준에서 성능이 포화됩니다. 따라서 "Point 수 상한 + 소형 인스턴스 컷오프"를 함께 적용해야 실효가 있다는 점을 실험으로 함께 확인드리겠습니다.

**중간 공유 7월 31일(금)(스크리닝 결과), 최종 8월 4일(화)** 목표입니다.

## 3. 4순위 — 유의미한 Instance의 최소 크기 기준 (진행 계획)

**9월 4일까지 진행 가능합니다.** 2순위의 러프 기준을 아래 두 축으로 고도화합니다.

- **클래스별 최소 크기 차등화**: 크기 구간별(size-bin) 클래스 AP 분석 + 클래스별 컷오프 스윕으로 19개 클래스 각각의 최소 유의미 크기를 도출합니다. 예비 분포 분석은 완료했으며, 대부분 클래스는 공통 기준(8px@1280)으로 충분하나 **rebar·small pipe 계열은 중앙값이 10px@1280 수준이라 별도(더 낮은) 하한이 필요**함을 확인했습니다.
- **겹침·절단·구김·파손 객체 기준**: 보이는 영역만 라벨링(가시 영역 기준)을 원칙으로 하되, 상태 플래그(겹침/절단 등) 부여안을 함께 검토합니다(COCO·PASCAL VOC 등의 crowd/difficult 처리 선례 준용). 기준 적용 전후의 모델 성능·라벨링 효율 변화도 함께 정리하겠습니다.

**중간 공유 8월 21일(금), 최종 9월 4일(금)** 목표입니다.
