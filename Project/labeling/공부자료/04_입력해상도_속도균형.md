# 04. 입력 해상도 ↔ 정확도/속도/메모리 — 3순위 근거자료

> 목적: 960/1280/1600/1920 중 dense 소형 scrap·4K·YOLO11/26-seg에 맞는 입력 크기 선택.
> 작성: 2026-06-03.

---

## 기술 토대 (검증 vs 추론)

- **[VERIFIED] 연산은 입력 한 변 길이에 ~제곱 비례.** conv 비용 ∝ 공간 차원 곱 → 한 변 2× → ~4× FLOPs(파라미터 불변). 960 기준 제곱비:
  - 1280 ≈ (1280/960)² ≈ **1.78×**
  - 1600 ≈ (1600/960)² ≈ **2.78×**
  - 1920 ≈ (1920/960)² ≈ **4.0×**
  (검증된 제곱 규칙으로부터의 산술 추론.) **activation 메모리도 동일 배율** → 1920은 960의 ~4× activation 메모리 = 학습 OOM 주원인, imgsz↑ 시 `batch`↓ 필요.
- **[VERIFIED] mAP는 해상도↑에 상승하나 체감 감소.** YOLOv4: 416→608서 AP 41.2%→43.5%(실재하나 둔화). 고해상 끝에서 정확도-FLOPs 효율 악화.
- **[INFERENCE] 우리 경우**: 1600→1920 한계 mAP 이득은 작을 것 — 단 **소형객체 recall이 병목이면 예외**. 3840 원본이면 1920도 2× 다운스케일 → 소형 scrap은 픽셀 손실. 소형 recall 병목 시 **tiling/SAHI가 imgsz↑보다 우월**할 수 있음(tile당 native 픽셀 밀도 유지).

## 참고문헌

**1. EfficientNet** — Tan & Le — ICML 2019 — https://arxiv.org/abs/1905.11946
depth·width·**해상도** 동시(compound) scaling. → 모델 용량 안 키우고 1920만 올리면 체감 감소. dense scrap엔 중간 크기 YOLO11 + 중간 해상도 조합.

**2. EfficientDet** — Tan et al. — CVPR 2020 — https://arxiv.org/abs/1911.09070
detection compound scaling, 깔끔한 accuracy–FLOPs Pareto(D0–D7). → **accuracy–latency trade-off 방법론의 정전.** 960/1280/1600/1920을 곡선 위 점으로 보고 기울기 평탄해지는 knee에서 정지.

**3. YOLOv4** — Bochkovskiy et al. — 2020 — https://arxiv.org/abs/2004.10934
입력 해상도 ablation(416/512/608). AP 41.2→43.0→43.5% = 해상도 단독 체감 감소 증거. 한 해상도서 학습→더 높은 해상도서 추론도 가능.

**4. Scaled-YOLOv4** — Wang et al. — CVPR 2021 — https://arxiv.org/abs/2011.08036
YOLO compound scaling(depth/width/해상도). 960→1920 ~제곱 budgeting 직접 지지.

**5. YOLOv7** — Wang et al. — CVPR 2023 — https://arxiv.org/abs/2207.02696
**640 vs 1280 입력에 별도 모델 변형**(E6/D6/E6E). → 1280은 640 위 표준 "고해상" tier. 4K scrap엔 1280–1600이 현실적 고해상 band, 1920은 E6E급 용량 필요.

**6. YOLOv9** — Wang et al. — ECCV 2024 — https://arxiv.org/abs/2402.13616
PGI+GELAN으로 정보 손실↓. 깊은 다운샘플서 소실되는 tiny scrap 특징에 의미 — 아키텍처 효율이 imgsz↑ 일부 대체.

**7. YOLO11 overview** — Khanam&Hussain — arXiv 2410.17725 + https://docs.ultralytics.com/models/yolo11/
C3k2+SPPF+**C2PSA self-attention**. docs: C2PSA가 **소형·가림 객체 검출 개선**(YOLOv8m 대비 ~22% 적은 param). → 우리 실제 모델, 소형 recall 일부 보완 → 1920 압박 완화. **1280 vs 1600 먼저 벤치마크.**

**8. Ultralytics `imgsz` docs** — https://docs.ultralytics.com/usage/cfg/
[VERIFIED] imgsz 기본 640, stride 배수(32)로 pad, 클수록 정확도↑·연산↑, GPU 메모리에 맞춰 튜닝, `multi_scale` 옵션. → 네 후보 모두 32 배수 OK. **`amp=True`(FP16)로 activation 메모리 ~절반**, 1600/1920 갈 때 batch 비례 축소.

**9. SNIP** — Singh&Davis — CVPR 2018 — https://arxiv.org/abs/1711.08189
CNN은 scale에 강건하지 않음; 소형 급격 저하. **객체 scale을 네트워크 scale에 맞추는 것이 raw 해상도보다 중요** → 네 크기에서 *AP_small*을 따로 측정하라.

**10. SAHI** — Akyon et al. — IEEE ICIP 2022 — https://arxiv.org/abs/2202.06934 + https://docs.ultralytics.com/guides/sahi-tiled-inference/
tile별 native scale 추론·merge → small AP +5–7%(추론만), +12–14%(sliced fine-tune). → **4K dense scrap 최고 레버리지 후보.** 1920에 ~4× 비용(여전히 2× 다운샘플) 대신 3840×2160을 ~960–1280 tile로. ※ native SAHI는 bbox 대상 → instance/panoptic엔 mask-aware tile merge 필요.

**11. ISDNet** — Guo et al. — CVPR 2022 — [VERIFIED](openaccess CVPR2022)
초고해상 segmentation의 메모리·연산 장벽; shallow(full-res)+deep(downsampled) 분기 융합. → 1920 brute-force 대안: 미세 디테일을 싸게 유지. 향후 custom 아키텍처 시 유용.

**12. HRNet** — Wang et al. — TPAMI 2020 — https://arxiv.org/abs/1908.07919
네트워크 전체에서 고해상 stream 유지. → 해상도를 *어디서* 보존하느냐가 *입력* 해상도보다 중요할 수 있음. 중간 imgsz로도 소형 seg 품질 가능.

## 결론 (사실 vs 추론 구분)
- **검증된 사실**: 연산·activation 메모리 ~제곱(960 대비 1280/1600/1920 ≈1.8/2.8/4.0×); 해상도 단독 mAP 체감 감소(YOLOv4); 1280이 표준 고해상 tier(YOLOv7/Ultralytics); tiling이 small AP↑(SAHI).
- **추론(우리 데이터로 검증 필요)**: 4K dense 소형 scrap은 **1280–1600 band**가 accuracy–latency knee 근처일 것. 1920은 ~4× 비용에 한계 mAP, batch 강제 축소. **결정적 실험 = imgsz별 AP_small/mask-AP·recall + ~1280 SAHI tiling head-to-head.** imgsz는 모델 용량에 맞출 것(1920 + nano 조합 금지).
