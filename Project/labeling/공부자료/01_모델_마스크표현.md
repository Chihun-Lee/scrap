# 01. Instance/Panoptic Segmentation 모델 & 마스크 표현 방식

> 조사 범위: dense(~200 instance)·중첩 철스크랩, 4K, 19클래스, polygon(LabelMe) 라벨, YOLO-seg(1280), PQ 평가.
> 모든 링크 web search로 검증 완료 (UNVERIFIED 없음). 작성: 2026-06-03.

---

## 1. Mask R-CNN — Kaiming He et al. — ICCV 2017 (Best Paper)
- https://arxiv.org/abs/1703.06870
- Faster R-CNN에 per-instance 마스크 분기 추가 + RoIAlign.
- **마스크 표현**: box 내부 고정 저해상도(보통 28×28) 마스크 → box 크기로 리사이즈.
- **시사점**: 28×28 in-box 마스크가 핵심 한계. dense 소형 객체에서 rebar/pipe/mesh 같은 thin 객체는 blur됨. detect-then-segment의 기준 baseline.

## 2. YOLACT — Bolya et al. — ICCV 2019
- https://arxiv.org/abs/1904.02689
- prototype 마스크 + per-instance 계수의 선형결합, box로 crop.
- **시사점**: **YOLO-seg 마스크 헤드의 직계 조상**. prototype은 28×28보다 고해상도라 경계가 선명하지만, 고정된 prototype 수(~32)로는 ~200 중첩 instance 표현에 한계. box 정확도 의존.

## 3. YOLACT++ — Bolya et al. — TPAMI 2020
- https://arxiv.org/abs/1912.06218
- deformable conv + mask re-scoring 추가.
- **시사점**: mask re-scoring은 dense 장면의 마스크 신뢰도 랭킹(→ PQ의 1:1 매칭)에 직접 유효.

## 4. SOLO — Wang et al. — ECCV 2020
- https://arxiv.org/abs/1912.04488
- box 없이 grid location으로 instance 직접 예측.
- **시사점**: box-free라 중첩 심한 scrap에서 box-NMS 억제 문제 회피. 단 중심 거의 겹치는 두 조각은 같은 cell 충돌.

## 5. SOLOv2 — Wang et al. — NeurIPS 2020
- https://arxiv.org/abs/2003.10152
- dynamic conv kernel + **Matrix NMS**.
- **시사점**: ~200 마스크에서 **mask 기반 NMS**가 box-NMS보다 중첩 처리 우수하고 저렴. full-res 마스크로 thin 경계 보존. YOLO-seg 대비 벤치마크 후보.

## 6. CondInst — Tian et al. — ECCV 2020
- https://arxiv.org/abs/2003.05664
- instance-conditioned dynamic conv, RoIAlign·고정 grid 없음.
- **시사점**: 28×28 병목 없이 고해상도 마스크. query/kernel 기반 마스크가 thin·dense에 유리한 이유의 가교.

## 7. PointRend — Kirillov et al. — CVPR 2020
- https://arxiv.org/abs/1912.08193
- 마스크 예측을 "렌더링"으로 보고 불확실(경계) point만 적응적 세분화.
- **시사점**: 마스크 **해상도** 결정에 직결. 고정 저해상도가 잃는 thin 경계를 point refinement로 복원 — PQ의 IoU 매칭에 유리.

## 8. Panoptic Segmentation (PQ 지표 논문) — Kirillov et al. — CVPR 2019
- https://arxiv.org/abs/1801.00868
- panoptic task 정의 + PQ 지표 도입.
- **시사점**: 우리 평가의 근간. **panoptic은 마스크 겹침 금지** — 물리적으로 겹치는 scrap을 depth/confidence로 평탄화해야 하며(6,8번 스크립트), 그 방식이 PQ에 직접 영향.

## 9. Panoptic FPN — Kirillov et al. — CVPR 2019
- https://arxiv.org/abs/1901.02446
- Mask R-CNN instance 분기 + semantic 분기 융합.
- **시사점**: "two-branch then merge" 표준 panoptic baseline. 우리 파이프라인(YOLO-seg→panoptic→PQ)과 개념 동일.

## 10. DETR — Carion et al. — ECCV 2020
- https://arxiv.org/abs/2005.12872
- transformer set prediction, anchor·NMS 없음. query에 mask head로 확장.
- **시사점**: NMS-free 패러다임의 시작 — ~200 중첩에 유리. 단 고정 query 수(~100)는 ~200 instance에 한계 → query 수 상향 필요.

## 11. MaskFormer — Cheng et al. — NeurIPS 2021
- https://arxiv.org/abs/2107.06278
- 모든 segmentation을 "mask classification"으로 재정의.
- **시사점**: 단일 모델이 native panoptic 출력. "클래스 수가 많을 때 per-pixel 대비 우수" — 우리 19클래스에 유리.

## 12. Mask2Former — Cheng et al. — CVPR 2022
- https://arxiv.org/abs/2112.01527
- masked attention + multi-scale 고해상도 feature. COCO 57.8 PQ.
- **시사점**: native panoptic·PQ 최강 후보(정확도 상한 벤치마크). 4K에서 무겁고 느림, query 수 ~200 대응 필요.

## 13. Mask DINO — Li et al. — CVPR 2023
- https://arxiv.org/abs/2206.02777
- DINO + mask 분기. panoptic 59.4 PQ.
- **시사점**: 현 transformer SOTA. anchor-box denoising query로 소형 객체 localization 우수. 정확도 기준점.

## 14. RTMDet — Lyu et al. — arXiv 2022
- https://arxiv.org/abs/2212.07784
- real-time detector, dynamic kernel mask head (SOLOv2/CondInst 계열).
- **시사점**: **YOLO-seg의 가장 직접적 real-time 경쟁자**. 28×28 병목 없이 빠름 → thin dense scrap 고해상도에 실용적 대안. PQ head-to-head 권장.

## 15. Ultralytics YOLO (YOLO26/11/v8) overview — Sapkota et al. — arXiv 2025
- https://arxiv.org/abs/2510.09653 / 아키텍처: https://arxiv.org/abs/2410.17725 / docs: https://docs.ultralytics.com/models/yolo11
- **우리 타깃 모델 계열**. 마스크 헤드는 **YOLACT식 prototype+계수**, box로 crop.
- **핵심 시사점**: (a) prototype이 stride-4 → 1280 입력 시 320×320, 640 대비 마스크 해상도 ~2배 ↑ (thin scrap·세밀 polygon 매칭에 중요); (b) box-crop이라 중첩 scrap은 box·box-NMS에 의존(dense pile 주 실패모드); (c) 고정 prototype 수(~32)가 분리 가능한 중첩 마스크 수 제한. ※ YOLOv8/11은 peer-review 논문 없음 → docs + YOLACT(2번) 공동 인용.

## 16. SAM (Segment Anything) — Kirillov et al. — ICCV 2023
- https://arxiv.org/abs/2304.02643
- promptable(point/box/mask) foundation 모델, class-agnostic 고해상도 마스크.
- **시사점**: panoptic·PQ 모델 아님. 활용: **라벨링 가속**(rough click → polygon급 마스크)·YOLO box 마스크 refinement.

## 17. SAM 2 — Ravi et al. — ICLR 2025
- https://arxiv.org/abs/2408.00714
- SAM의 image+video 확장, 더 빠르고 정확.
- **시사점**: 정지 4K scrap에선 "더 좋은 SAM"으로 **annotation/refinement 부트스트랩**용. video propagation은 무관.

## PQ (Panoptic Quality) 정의 — 우리 평가지표
PQ는 클래스별 계산 후 평균. 예측·GT segment 매칭에서 **IoU > 0.5만 TP**(고유 매칭 보장).

**PQ = (Σ_TP IoU) / (TP + ½FP + ½FN) = SQ × RQ**
- **SQ(Segmentation Quality)** = 매칭된 TP의 평균 IoU (마스크 형태 품질)
- **RQ(Recognition Quality)** = TP/(TP+½FP+½FN), F1형 (검출/카운팅 품질)

→ 정확히 검출된 segment의 tight 경계를 보상, 누락(FN)·중복/허위(FP)·경계 부정확(낮은 SQ)을 처벌. **객체 크기 무관, 클래스 균등 가중.**
→ **우리 dense·thin·중첩 scrap에 함의**: IoU>0.5 hard threshold가 thin 객체(rebar/pipe/mesh)에 가혹 — 수 px 경계 오차로 0.5 미달 시 near-miss가 FP+FN 동시 발생. 따라서 **마스크 해상도(1280 입력, prototype/dynamic-conv 고해상도 head, PointRend/SAM refinement)**와 **instance→panoptic 변환 시 깔끔한 겹침 해소**가 PQ 최고 레버리지.
